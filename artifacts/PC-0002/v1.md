# Event-Driven Thinking in Software Design

Most systems are built around state. Mature systems are built around events.

## State vs Event

- State answers: *What is true now?*
- Events answer: *What happened?*

Event-driven systems shift the center of gravity from current state to historical causality.

## Benefits

- full auditability
- replayable system behavior
- natural scalability in distributed systems

## Design Shift

Instead of updating state directly, systems should:
1. emit events
2. derive state from events
3. treat logs as first-class data

## Conclusion

If you cannot reconstruct your system from its events, you do not have a durable architecture.
